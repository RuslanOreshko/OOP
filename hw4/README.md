# Домашня робота

## Factory vs AbstractFactory. Різниця, переваги та мінуси

### Різниця між Factory та AbstractFactory

Factory:

- Звичайний фабричний метод виористовується для створення одного типу продукту.
- Клієнт не знає точний клас продукту, лише інтерфес або базовий клас.

Abstract Factory:

- Створює сімейство взяємопов'язаний продуктів.
- клієнт працює лише через інтерфейс.
- фабрика контролює сумісність продуктів(Dev/prod). Приклад:

```csharp
public class ProdFactory : IAppFactory
{
    private ILogger? logger;
    private IDatabase? database;

    public ILogger CreateLogger()
    {
        if(logger == null)
        {
            logger = new FileLogger();
        }
        return logger;
    }

    public IDatabase CreateDatabase()
    {
        if(database == null)
        {
            database = new SqlDatabase();
        }
        return database;
    }

}
```

### Переваги та мінуси

Factory:

- Перваги:
  - Інкапсуляція створення об'єктів.
  - Менша залежність від конкретної реалізації.
  - Легка заміна конкретного класу продукту
- Мінуси:
  - Кожна нова конкретна реалізація продукту потребує нового кнокретного творця.
  - Може бути багато класів.

Abstract Factory:

- Перваги:
  - Сумісність продуктів одного сімейства.
  - Можна легко поміняти середовище через одну фабрику.
  - Клієнт зовсім не бачить конкретних реалізацій.
- Мінуси:
  - Може бути дуже велике дерево (сімейство продуктів та фабрика).
  - Трохи складніша підтримка ні проста фабрика.
  - Додатковий рівень абстракції.

### Висновки

Тобто коли потрібно контролювати створення лише одного типу продукту, або коли клієнт повинен працювати через абстракцію, то краще буде використати звичайний фабричний метод.
Якщо Потрібно створити декілька сумістних продуктів одночасно, або якщо потрібно що б програма підтримувала декілька середовищ, то краще використовувати абстрактний фабричний патерн.

Приклад Абстрактної фабрики та звичайного фабричного методу можна подивитись в проєкті, у файлах `AbstractFactory.cs` та `Factory.cs`.
